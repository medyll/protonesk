# S4-01 — Config multi-compte dans config.yaml

**Sprint:** 4 | **Effort:** S | **Status:** planned

## User Story

En tant qu'utilisateur, je veux déclarer plusieurs comptes Proton dans `config.yaml` sans modifier le code.

## Acceptance Criteria

- [ ] `config.yaml` accepte clé `accounts` (liste de dicts)
- [ ] Chaque compte : `username`, `label` (optionnel, défaut = username)
- [ ] `load_config()` retourne `accounts` dans le dict mergé
- [ ] Rétrocompatible : si `accounts` absent → comportement actuel (compte unique via keyring)
- [ ] Validation : si `accounts` présent mais vide → erreur explicite au démarrage

## Exemple config

```yaml
accounts:
  - username: perso@proton.me
    label: perso
  - username: pro@proton.me
    label: pro
```
