# S2-03 — Mapping IMAP ↔ Proton

**Sprint:** 2 | **Effort:** M | **Dépend de:** S2-02, S1-02 (api_client)

## User Story

En tant qu'utilisateur, je veux que mes dossiers Proton (INBOX, Sent, Trash, Archive) apparaissent correctement dans mon client IMAP, avec le bon nombre de messages et les bons flags.

## Acceptance Criteria

- [ ] `LIST` retourne les dossiers Proton mappés en mailboxes IMAP
- [ ] `SELECT INBOX` retourne `EXISTS` (nb messages) et `UNSEEN` count corrects
- [ ] Labels Proton → noms IMAP standards (`INBOX`, `Sent`, `Trash`, `Drafts`, `Archive`)
- [ ] Flag Proton `Unread=1` → IMAP flag `\Seen` absent
- [ ] Message ID Proton → séquence IMAP numérique (1, 2, 3...)
- [ ] Cache des metadata en mémoire (TTL 60s) pour éviter appels API répétés

## Mapping dossiers

| Label Proton | Mailbox IMAP |
|---|---|
| INBOX | INBOX |
| Sent | Sent |
| Trash | Trash |
| Drafts | Drafts |
| Archive | Archive |
| Spam | Junk |
| Labels custom | label-name |

## Implémentation cible

- `src/imap_bridge.py` — classe `ProtonIMAPBridge`
- Méthodes : `get_mailboxes()`, `get_mailbox_status()`, `get_message_list()`, `message_id_to_seq()`
- Cache dict en mémoire avec timestamp
