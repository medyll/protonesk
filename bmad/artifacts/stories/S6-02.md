# S6-02 — Service systemd Linux

**Sprint:** 6 | **Effort:** S | **Status:** planned

## User Story

En tant qu'utilisateur Linux, je veux que le bridge tourne comme service systemd au démarrage.

## Acceptance Criteria

- [ ] `scripts/proton-bridge.service` — unit file systemd valide
- [ ] `scripts/install-service-linux.sh` copie unit, reload daemon, enable service
- [ ] `Restart=on-failure`, `RestartSec=5s`
- [ ] `WantedBy=default.target` (user service, pas root)
- [ ] Tourne en user service (`systemctl --user`) pour accès keyring
- [ ] Logs via `journald` (`journalctl --user -u proton-bridge -f`)
- [ ] `uninstall` : disable + stop + supprime unit file

## Unit file cible

```ini
[Unit]
Description=Proton Mail Bridge
After=network.target

[Service]
Type=simple
ExecStart=/usr/bin/python3 /opt/proton-bridge/main.py
WorkingDirectory=/opt/proton-bridge
Restart=on-failure
RestartSec=5s
Environment=PYTHONUNBUFFERED=1

[Install]
WantedBy=default.target
```

## Usage

```bash
./scripts/install-service-linux.sh install
systemctl --user start proton-bridge
systemctl --user status proton-bridge
journalctl --user -u proton-bridge -f
```
