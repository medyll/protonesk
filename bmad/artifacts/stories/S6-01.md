# S6-01 — Service Windows (NSSM)

**Sprint:** 6 | **Effort:** M | **Status:** planned

## User Story

En tant qu'utilisateur Windows, je veux que le bridge démarre automatiquement avec Windows sans garder un terminal ouvert.

## Acceptance Criteria

- [ ] `scripts/install-service-windows.ps1` installe le bridge comme service Windows
- [ ] Télécharge NSSM si absent (`%LOCALAPPDATA%\ProtonBridge\nssm.exe`)
- [ ] Service name : `ProtonMailBridge`, display name : `Proton Mail Bridge`
- [ ] Start type : Automatic (démarre avec Windows)
- [ ] Tourne sous compte utilisateur courant (requis pour accès keyring)
- [ ] Logs vers `%LOCALAPPDATA%\ProtonBridge\logs\bridge.log` (rotation 10 Mo)
- [ ] Commandes : `install`, `uninstall`, `start`, `stop`, `status`
- [ ] `uninstall` arrête + supprime service + laisse config/logs intacts

## Usage

```powershell
.\scripts\install-service-windows.ps1 install
.\scripts\install-service-windows.ps1 start
.\scripts\install-service-windows.ps1 status
.\scripts\install-service-windows.ps1 stop
.\scripts\install-service-windows.ps1 uninstall
```

## Contraintes

- NSSM téléchargé depuis GitHub releases (hash vérifié)
- Service interactif requis (keyring nécessite session utilisateur active)
- Python path détecté automatiquement (`where.exe python`)
- Working directory = répertoire du projet
