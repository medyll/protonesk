# S6-04 — Tray icon Windows

**Sprint:** 6 | **Effort:** L | **Status:** planned

## User Story

En tant qu'utilisateur Windows, je veux voir l'état du bridge dans la barre système et le contrôler sans terminal.

## Acceptance Criteria

- [ ] `src/tray.py` — icône system tray via `pystray`
- [ ] Icône colorée selon état : vert (connecté), rouge (erreur), gris (arrêté)
- [ ] Menu tray :
  - Statut : `✅ Connecté — 2 comptes` ou `❌ Erreur`
  - `Ouvrir config.yaml`
  - Séparateur
  - `Démarrer` / `Arrêter` bridge
  - `Relancer` (restart daemon)
  - Séparateur
  - `Quitter`
- [ ] Notification Windows (toast) si nouvelle connexion client IMAP
- [ ] Notification si session Proton expirée + reconnexion en cours
- [ ] `main.py --tray` démarre tray + bridge dans le même process (threads séparés)
- [ ] Tray seul sans `--tray` : comportement actuel (headless)

## Architecture

```
main.py --tray
    ├── asyncio thread : IMAP + SMTP servers (inchangé)
    └── pystray thread : tray icon + menu
            ↕ communication via threading.Event + queue
```

## Contraintes

- `pystray` requis (`pip install pystray pillow`)
- Icônes : PNG 16x16 et 32x32 dans `assets/icons/`
- Windows uniquement pour les toasts (via `win10toast` ou `plyer`)
- Thread pystray bloquant — asyncio dans thread séparé via `asyncio.run()` dans `threading.Thread`
- Si `pystray` absent → `--tray` affiche warning et démarre headless
