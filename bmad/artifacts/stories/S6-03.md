# S6-03 — Script install cross-platform

**Sprint:** 6 | **Effort:** M | **Status:** planned

## User Story

En tant que nouvel utilisateur, je veux installer le bridge en une commande sans connaître l'architecture interne.

## Acceptance Criteria

- [ ] `scripts/install.py` — détecte OS (Windows/Linux/macOS), appelle bon installeur
- [ ] Vérifie Python >= 3.11, affiche erreur claire si trop vieux
- [ ] `pip install -r requirements.txt` automatique si packages manquants
- [ ] Installe `proton-python-client` depuis GitHub si absent
- [ ] Génère `config.yaml` interactif si absent (ports, password local, TLS y/n)
- [ ] Lance `python src/secrets.py setup` si credentials absents du keyring
- [ ] Affiche résumé post-install : ports, commande de démarrage, config Thunderbird
- [ ] `--uninstall` : supprime service + optionnellement config/credentials

## Flow install

```
python scripts/install.py
  → Détect OS
  → Check Python version
  → pip install deps
  → Génère config.yaml (interactif)
  → python src/secrets.py setup (interactif, credentials keyring)
  → Install service (Windows: NSSM / Linux: systemd)
  → Start service
  → Affiche résumé connexion IMAP/SMTP
```

## Résumé post-install

```
✅ Protonesk installé et démarré

Connexion IMAP :  127.0.0.1:1143  (mot de passe: bridge)
Connexion SMTP :  127.0.0.1:1025

Thunderbird : Ajouter compte → IMAP manuel → 127.0.0.1:1143
```

## Contraintes

- Pas de dépendance externe au-delà de stdlib Python pour le script install lui-même
- Windows : doit tourner depuis PowerShell non-admin (NSSM user service)
- macOS : service via `launchd` (LaunchAgent) — implémentation minimale, non bloquante
