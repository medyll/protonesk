# S2-04 — FETCH message avec déchiffrement PGP inline

**Sprint:** 2 | **Effort:** M | **Dépend de:** S2-02, S2-03, S1-03 (crypto)

## User Story

En tant qu'utilisateur, je veux que mon client IMAP puisse télécharger le contenu complet de mes mails Proton, déjà déchiffrés, comme si c'était des mails normaux.

## Acceptance Criteria

- [ ] `FETCH <seq> (RFC822)` retourne message complet en format RFC 822 (headers + body)
- [ ] `FETCH <seq> (BODY[HEADER])` retourne seulement les headers
- [ ] `FETCH <seq> (BODY[TEXT])` retourne body déchiffré en texte plain
- [ ] Déchiffrement PGP via `ProtonCrypto.decrypt_message_body()` (in-memory, pas de fichier)
- [ ] Body HTML converti en text/plain via `ContextFormatter.html_to_markdown()`
- [ ] Si déchiffrement impossible → retourne `[Encrypted message - decryption failed]`
- [ ] Marque message comme lu après FETCH complet (configurable)

## Format RFC 822 cible

```
From: sender@proton.me
To: recipient@example.com
Subject: Test email
Date: Thu, 07 May 2026 10:00:00 +0000
Content-Type: text/plain; charset=utf-8

Corps du message déchiffré ici.
```

## Implémentation cible

- Méthode `fetch_message(seq, parts)` dans `ProtonIMAPBridge`
- Appelle `ProtonAPIClient.get_message()` → `ProtonCrypto.decrypt()` → format RFC 822
- Intégré dans handler `FETCH` de `ProtonIMAPServer`

## Tests requis

- Test format RFC 822 output (headers présents, body déchiffré)
- Test FETCH HEADER uniquement (pas de body fetch)
- Test déchiffrement échoué → message placeholder
- Test mock crypto → body plaintext correct
