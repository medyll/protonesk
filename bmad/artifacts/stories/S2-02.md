# S2-02 — Serveur IMAP core (asyncio)

**Sprint:** 2 | **Effort:** L | **Dépend de:** S1-01 (ProtonAuth)

## User Story

En tant qu'utilisateur, je veux connecter Thunderbird ou tout client IMAP à `localhost:1143` et voir ma boîte Proton comme si c'était un serveur IMAP standard.

## Acceptance Criteria

- [ ] Serveur écoute sur `127.0.0.1:1143` (configurable)
- [ ] Implémente commandes IMAP4 minimales : `CAPABILITY`, `LOGIN`, `SELECT`, `LIST`, `FETCH`, `LOGOUT`
- [ ] Auth IMAP acceptée avec credentials locaux (définis dans config)
- [ ] Répond aux commandes avec format RFC 3501 valide
- [ ] Gestion concurrente de plusieurs connexions (asyncio)
- [ ] Timeout connexion inactive après 30 min

## Commandes IMAP à implémenter (RFC 3501 subset)

```
CAPABILITY  → liste les capacités
LOGIN       → authentifie client (local check, pas Proton)
LIST        → liste les mailboxes/dossiers
SELECT      → sélectionne une mailbox
EXAMINE     → idem SELECT mais read-only
FETCH       → récupère messages (headers + body)
STORE       → modifie flags (\Seen, \Deleted)
LOGOUT      → ferme session
NOOP        → keepalive
```

## Implémentation cible

- `src/imap_server.py` — classe `ProtonIMAPServer` (asyncio StreamReader/Writer)
- Parser de commandes IMAP (ligne par ligne, format `TAG CMD ARGS`)
- Dispatcher vers handlers par commande

## Tests requis

- Test connexion TCP et greeting `* OK`
- Test séquence `LOGIN → SELECT INBOX → LOGOUT`
- Test commande inconnue → `BAD`
- Test format réponse CAPABILITY
