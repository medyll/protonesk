# S2-05 — Daemon entry point (main.py)

**Sprint:** 2 | **Effort:** S | **Dépend de:** S2-01, S2-04

## User Story

En tant qu'utilisateur, je veux lancer le bridge avec une seule commande et avoir IMAP + SMTP disponibles immédiatement, sans configuration complexe.

## Acceptance Criteria

- [ ] `python main.py` démarre les deux serveurs (IMAP:1143 + SMTP:1025)
- [ ] Authentification Proton au démarrage via `ProtonAuth` (keyring)
- [ ] Log de démarrage clair : ports, statut connexion Proton
- [ ] `Ctrl+C` arrête proprement (logout Proton + fermeture serveurs)
- [ ] `python main.py --imap-port 1143 --smtp-port 1025` (ports configurables CLI)
- [ ] `python main.py --smtp-only` ou `--imap-only` (démarrage partiel)
- [ ] Reconnexion auto si session Proton expire (retry 3x avec backoff)

## Log de démarrage attendu

```
🔐 Authenticating with Proton...
✅ Connected as user@proton.me
📬 IMAP server started on 127.0.0.1:1143
📤 SMTP server started on 127.0.0.1:1025
🚀 Proton Bridge ready. Press Ctrl+C to stop.
```

## Implémentation cible

- `main.py` à la racine du projet
- `argparse` pour options CLI
- `asyncio.gather()` pour démarrer les deux serveurs en parallèle
- Signal handler `SIGINT` pour shutdown propre

## Tests requis

- Test démarrage serveurs (mock servers)
- Test shutdown handler (mock SIGINT)
- Test reconnexion auto (mock session expire)
- Test args CLI (imap-port, smtp-port, imap-only, smtp-only)
