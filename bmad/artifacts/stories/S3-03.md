# S3-03 — IMAP IDLE

**Sprint:** 3 | **Effort:** M | **Dépend de:** S2-02

## Acceptance Criteria

- [ ] `IMAPSession` gère commande `IDLE`
- [ ] Après `IDLE` → serveur répond `+ idling`, reste en attente
- [ ] Background task poll `get_mailbox_status()` toutes les 30s
- [ ] Si EXISTS change → pousse `* N EXISTS` au client
- [ ] Client envoie `DONE` → IDLE terminé, session reprend normalement
- [ ] Timeout IDLE après 29 min (RFC recommande < 30 min)
