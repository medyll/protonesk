# Sprint 2 — IMAP/SMTP Bridge Layer

**Goal:** Exposer Proton Mail via protocoles IMAP/SMTP locaux — tout client mail ou agent AI peut s'y connecter comme à n'importe quel serveur mail.

**Date:** 2026-05-07

---

## Stories

| ID | Titre | Effort | Dépendances |
|----|-------|--------|-------------|
| S2-01 | Serveur SMTP local (réception → Proton send) | M | S1-04 |
| S2-02 | Serveur IMAP core (asyncio, commandes LIST/SELECT/FETCH) | L | S1-01 |
| S2-03 | Mapping IMAP ↔ Proton (dossiers, flags, metadata) | M | S2-02 |
| S2-04 | Fetch message IMAP avec déchiffrement PGP inline | M | S2-02, S2-03 |
| S2-05 | Daemon entry point — main.py (démarre IMAP+SMTP, gère session) | S | S2-01, S2-04 |

**Effort total:** ~3-4 jours dev. Capacité estimée: 1 sprint.

## Architecture cible

```
Client (Thunderbird / Claude / n8n)
        ↓ IMAP:1143        ↓ SMTP:1025
   IMAPServer           SMTPServer
   (asyncio)            (aiosmtpd)
        ↓                    ↓
   ProtonAPIClient      ProtonSend
   + ProtonCrypto       + ProtonCrypto
        ↓                    ↓
            Proton REST API
```

## Dépendances à ajouter

- `aiosmtpd>=1.4.0` — serveur SMTP moderne (remplace `smtpd` déprécié)
- Pas de lib externe pour IMAP server — implémentation asyncio custom (protocole IMAP4 RFC 3501)

## Out of scope (Sprint 2)

- TLS/SSL sur IMAP/SMTP (ajout Sprint 3)
- Authentification multi-compte
- IDLE IMAP (push notifications)
- Port 993/465 (SSL natif)
